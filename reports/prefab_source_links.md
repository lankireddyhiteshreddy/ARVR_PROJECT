# Prefab source links and instructions

This file helps you find where each group of prefabs likely came from on the internet (Asset Store, GitHub, creator pages).
Use it as a checklist to show your teacher where assets originated and to gather license/attribution info.

Files to open first:
- `reports/prefab_sources.md` — grouped used prefabs by top-level folder (already generated).
- `reports/scene_map.md` — shows which scenes use which scripts / prefabs.

How to use this file
1. Open `reports/prefab_sources.md` and find the asset group you want to show (example: Brick Project Studio).
2. Click or paste one of the search links below into your browser. The Google search includes keywords that tend to find asset store or GitHub pages.
3. From the asset page, copy the publisher/author name and the license/terms. If the folder contains a `README.md` or `LICENSE` file, open it and include that in your evidence.
4. When your teacher asks, present:
   - The `reports/prefab_sources.md` entry for the package (sample prefab paths)
   - The asset store/GitHub link you found
   - Any README/LICENSE file from `Assets/<package>/` if present

Tips when showing your teacher
- Show the `reports/prefab_sources.md` and open one or two example prefabs in the Unity Editor (right-click → Show in Explorer) to display the folder structure and any README/LICENSE files the package included.
- Use the Google or Asset Store link to show the original asset page and point out the publisher and license.
- If you used an asset in a way that requires attribution, copy the license text or the publisher's instructions and include them with the submission.
